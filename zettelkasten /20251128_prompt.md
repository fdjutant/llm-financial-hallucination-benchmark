# UK/EU LLM Financial Truthfulness Benchmark – 6-Week Plan
A structured, research-grade project plan designed to demonstrate skills in:
AI evaluation, safety, ML research engineering, data analysis, and cloud data engineering.

---

## 📌 Project Title
**Benchmarking Hallucinations in LLMs on Fact-Based Financial Questions from UK/EU Regulatory Filings**

---

## 🎯 Project Goal
Build a reproducible and rigorous pipeline that:
1. Extracts ground-truth financial facts from UK/EU company filings  
2. Auto-generates fact-based financial Q&A pairs  
3. Evaluates LLM truthfulness and hallucinations  
4. Produces analysis, plots, and a research-style write-up  
5. Optionally deploys on GCP for automation and scale  

This project is portfolio-ready and specifically tailored for AI research engineer roles in the UK.

---

# 📅 6-Week Execution Plan

---

# **WEEK 1 — Data Acquisition & Financial Fact Extraction**
**Deliverables:** Raw financial filings + structured facts table

### Objectives
- Download UK/EU financial filings  
- Extract structured financial metrics  
- Build a clean, well-defined dataset

### Tasks
- Select 5–10 companies (FTSE 100 recommended):
  - BP, Shell, HSBC, Barclays, Unilever, AstraZeneca, Vodafone, Diageo  
- Download filings from:
  - Companies House (iXBRL)  
  - LSE RNS announcements  
  - Issuer annual reports  
- Extract metrics:
  - Revenue  
  - Profit before tax  
  - Net income  
  - Assets & liabilities  
  - Operating cash flow  
- Create `facts.csv`:
  ```
  company, ticker, metric_name, value, unit, period_end, source_url
  ```

### Milestone
✔ Structured UK/EU financial facts for 50–150 data points.

---

# **WEEK 2 — Create Ground-Truth Q&A Set**
**Deliverables:** Templated Q&A dataset with authoritative answers

### Objectives
- Convert facts → templated Q&A  
- Ensure answers are verifiable and numeric  

### Tasks
- Build templates:
  - “What was {company}’s revenue in {year}?”
  - “How much profit before tax did {company} report for {period_end}?”
- Auto-generate 500–1,500 Q&A pairs  
- Define labeling rules:
  - Correct < 2% error  
  - Approx 2–10%  
  - Wrong > 10%  
  - Hallucination: fabricated info or wrong entity/year  

### Milestone
✔ `questions.csv` with hundreds of high-quality Q&A pairs.

---

# **WEEK 3 — LLM Evaluation Harness**
**Deliverables:** Automated evaluation pipeline

### Objectives
- Query models at scale  
- Capture, parse, and classify outputs  

### Tasks
- Implement evaluation script to:
  - Load questions  
  - Query GPT or open-source models  
  - Log raw answers  
- Parse numeric values (units, commas, currency)  
- Assign correctness labels:
  ```
  q_id, model_name, answer_raw, answer_parsed, error, classification
  ```
- Run multiple LLMs for comparison  

### Milestone
✔ Automated truthfulness scoring for each model.

---

# **WEEK 4 — Analysis & Failure Mode Breakdown**
**Deliverables:** Research insights + visualizations + taxonomy

### Objectives
- Quantify LLM weaknesses  
- Identify and categorize hallucination patterns  

### Tasks
- Compute:
  - Accuracy per model  
  - Accuracy per metric (revenue, PBT, etc.)  
  - Accuracy per firm  
  - Error distributions  
- Create visualizations:
  - Heatmaps  
  - Boxplots  
  - Failure-rate charts  
- Build a “Failure Mode Taxonomy”:
  - Wrong year  
  - Wrong metric  
  - Wrong scale (Millions vs Thousands)  
  - Fictional numbers  
  - Overconfident hallucinations  

### Milestone
✔ Jupyter notebook with analysis and visualisations.

---

# **WEEK 5 — Engineering Polish (Optional: Cloud/GCP Integration)**
**Deliverables:** Professional-grade infrastructure

### Objectives

### Milestone
✔ Cloud-ready research pipeline demonstrating ML + Data Engineering.
