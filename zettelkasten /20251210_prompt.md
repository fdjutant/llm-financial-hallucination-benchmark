You are implementing an end-to-end ingestion pipeline for **inline-XBRL embedded in HTML**.
Your job is to build four functions, each producing one table in the canonical data framework.
Follow all instructions precisely and do not skip steps.

You must implement the following tables in order:

---

## **1. Filing-Level Table (`filing`)**

This table contains **one row per filing** (one per HTML document set).
The inline-XBRRL HTML files are stored in ./data/raw/FCA/*/*.html

### **Your function must extract and populate:**

* `filing_id` (string; a stable identifier you derive, e.g., filename or user-provided ID)
* `company_number` (if available inside the HTML metadata; else null)
* `company_name` (if available from ix:entity or embedded document metadata)
* `period_start` (from contexts)
* `period_end` (from contexts)
* `balance_sheet_date` (if present)
* `accounting_standard` (FRS 102, IFRS, etc., infer from taxonomy prefixes)
* `source_type`

  * always `"inline_ixbrl_html"` for now
* `original_file_paths` (list of HTML fragments)
* `load_timestamp` (UTC now)

### **Your function must:**

* Scan `<ix:header>` and `<ix:entity>` sections if present
* When metadata is not explicitly supplied, infer **period_start** and **period_end** using the earliest and latest `<context>` periods extracted in step 2
* Output a dictionary or DataFrame with exactly one row

---

## **2. Context Table (`context`)**

This table stores **every XBRL `<context>`** found in the inline HTML.

### **Your function must extract:**

For each `<context id="...">`:

* `context_id` (the raw @id attribute)
* `filing_id` (fk)
* `entity_identifier` (from `<entity><identifier>`)
* `entity_scheme`
* `period_type` (`"instant"` or `"duration"`)
* `instant` **or** `period_start`, `period_end`
* `dimensional_qualifier`

  * a JSON object representing `<segment>` or `<scenario>` content
  * include any typed members or explicit dimensions

### **Your function must:**

* Handle contexts embedded within multiple HTML fragments (merge)
* Preserve **all** contexts, even if unused by facts
* Normalise dates to ISO format
* Return a DataFrame or list of rows

---

## **3. Raw Fact Table (`fact_raw`)**

This table stores **every tagged inline-XBRL fact** BEFORE interpretation.

### **Your function must extract for each fact:**

* `fact_id` (unique ID you generate)
* `filing_id`
* `context_id`
* `raw_name` (the QName, e.g., `esef_cor:Revenue`, `ifrs-full:Assets`)
* `taxonomy_domain`

  * prefix before colon (`ifrs-full`, `esef_cor`, `frs-core`, etc.)
* **If numeric (`ix:nonFraction`):**

  * `unit_ref`
  * `measure` (resolve via `<unit>` table inside the document)
  * `decimals`
  * `value_numeric` (normalised float; remove thousands separators)
* **If non-numeric (`ix:nonNumeric`):**

  * `value_text` (clean text content)
* `data_type` (`numeric` or `string`)
* `source_anchor` (CSS selector, absolute xpath, or fragment ID for traceability)

### **Your function must:**

* Parse all `<ix:nonFraction>` and `<ix:nonNumeric>` tags
* Correctly resolve escaped values (`ix:escaped="true"`)
* Extract units from `<unit id="...">`
* Fully handle nested tags and inline formatting
* Produce a DataFrame where each row is one fact

---

## **4. Canonical Facts Table (`fact_canonical`)**

This table contains **interpreted, structured, cleaned facts** used for downstream Q&A and hallucination benchmarking.

### **Your function must transform `fact_raw` into:**

* `canonical_fact_id`
* `filing_id`
* `source_fact_ids` (all contributing raw fact IDs)
* `concept_key`

  * a normalised semantic label (e.g., `BS.total_assets`, `PL.revenue`, `DISC.employee_count`)
  * derived via taxonomy lookup + heuristic mapping you design
* `taxonomy_domain`
* `statement_type`

  * `balance_sheet`, `income_statement`, `cash_flow`, `notes`, `directors_report`, `other`
* Time semantics

  * `period_type` (`instant` or `duration`)
  * `as_of_date` (if instant)
  * `period_start`, `period_end` (if duration)
  * Optionally derive `fy_label` (`current_fy`, `previous_fy`, etc.)
* Value

  * `value_numeric` OR `value_text`
  * `unit` (if applicable)
* Quality flags

  * `is_primary` (not derived)
  * `is_consistent` (true if matches roll-ups)

### **Your function must:**

* Resolve dimensional qualifiers into meaningful structure (e.g., class of PPE, type of share capital)
* Deduplicate identical facts reported in multiple HTML fragments
* Detect and resolve common IFRS label variants
* Produce a canonical fact table suitable for benchmarking LLM reasoning
